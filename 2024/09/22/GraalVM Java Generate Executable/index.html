GraalVM 生成可执行程序（Windows，乱码解决）

【参考文章】

【1】：Installation on Windows Platforms

【2】：https://github.com/oracle/graal/issues/6724

【3】：https://github.com/oracle/graal/issues/2492

【4】：https://docs.oracle.com/en/java/javase/22/intl/supported-encodings.html

环境准备：Java，Visual Studio Build Tools and Windows SDK，Maven等，可以参考 Installation on Windows Platforms

  • 当我学习 Spring 时，接触到了一个东西叫做 GraalVM 用于生成可执行的 Java 程序
  • 这个东西也可以用在普通的 jar 包，或者 class 文件上

普通 class 文件编成 exe

创建 java 源文件

  • 这一步非常重要，源文件要用 ASCII 码进行保存

编译源码，指定格式：javac -encoding GBK Main.java

在开始菜单打开 x64 Native Tools Command Prompt for VS xxxx 进入 native 环境

切换到 Main.class 所在的目录

输入命令： native-image -J"-Dfile.encoding=GBK" Main ，Main 是你的 class 文件名，注意有双引号

一定要指定编码方式为 GBK 不然运行中文会有乱码问题

运行，没问题

jar 包文件编成 exe

这里使用 Maven，下面是 pom.xml 文件的一部分配置，可以参考

主类的内容：Main.java

到 pom.xml 文件目录下运行命令：mvn clean package

此时会生成一个 jar 包，例如：Main.jar

先用 java -jar Main.jar 运行一下看看有没有问题，再进行下一步，如果显示缺少主类请看下面的方法

用压缩包程序打开 jar 包，打开目录 META-INF 找到 MANIFEST.MF 文件，编辑，在里面手动指定主类，即指定 main 方法所在的类：比如我的 main 方法写在类 org.wind.Main 那么在 MANIFEST.MF 添加以下这一行：

Main-Class: org.wind.Main

再次运行，应该没有无主类的问题了，如果有其他问题，再上网找，这里不再赘述

接下来，同样的，在开始菜单打开 x64 Native Tools Command Prompt for VS xxxx 进入 native 环境

切换到 jar 包所在目录，运行命令：native-image -J"-Dfile.encoding=GBK" -jar .\Main.jar 注意有双引号

一定要指定编码方式为 GBK 不然运行中文会有乱码问题

运行，没问题

如果在 Spring 上时，解决乱码方法很简单，Maven 打包时指定使用 native profile 就可以了

运行aot提前处理命令：mvn springboot:process-aot

运行native打包：mvn -Pnative native:build
