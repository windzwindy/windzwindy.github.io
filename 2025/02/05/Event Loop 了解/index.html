Event Loop 了解

  • 事件循环（Event Loop）是 JavaScript 中处理异步操作的机制。

  • 它允许 JavaScript 在单线程环境中执行非阻塞的代码，从而实现异步编程。事件循环的工作原理涉及多个组件，包括调用栈、任务队列（宏任务队列和微任务队列）以及事件循环本身。

主要组件

  • 调用栈（Call Stack）：
    • JavaScript 是单线程的，所有的代码执行都在调用栈中进行。调用栈是一个后进先出（LIFO）的数据结构，用于管理函数调用。当一个函数被调用时，它会被推入栈中；当函数执行完毕时，它会从栈中弹出。
  • 任务队列（Task Queue）：
    • 任务队列用于存放宏任务（macrotasks），例如 setTimeout、setInterval、I/O 操作等。宏任务队列中的任务会在调用栈为空时依次执行。
  • 微任务队列（Microtask Queue）：
    • 微任务队列用于存放微任务（microtasks），先进先出，例如 Promise 的回调和 queueMicrotask。微任务的优先级高于宏任务，微任务会在当前执行栈清空后、下一个宏任务开始之前执行。

工作流程

1. 执行全局代码，函数调用被推入调用栈。
2. 当遇到异步操作（如 `setTimeout` 或 Promise）时，相关的回调会被放入相应的任务队列。
3. 当调用栈为空时，事件循环会首先检查微任务队列，执行所有微任务，先进先出。
4. 微任务执行完后，事件循环会从宏任务队列中取出下一个任务并执行。
5. 重复以上步骤，直到所有任务都被处理完。
