Xorg（图形显示服务器）

  • Xorg 是一个开源的图形显示服务器，用于类 Unix 操作系统（如 Linux）上，它是 X Window System 的实现之一。Xorg 提供了一个图形化界面，使得用户能够通过鼠标和键盘与计算机进行交互，通常它用于管理显示、输入设备（如鼠标、键盘）和图形渲染。
  • 在 Xorg 中，X 服务器（X Server）是核心组件，它负责显示和管理窗口，同时与硬件（显示器、输入设备等）进行交互。而 X 客户端（X Client）是运行在 X 服务器上的程序，它们请求 X 服务器进行图形操作。常见的 X 客户端包括桌面环境（如 GNOME、KDE）和应用程序。
  • 简而言之，Xorg 是让图形化桌面环境和程序能够在 Linux 或其他类 Unix 系统中运行的重要基础设施。

Wayland（现代的显示服务器）

  • 文档

  • Wayland 是一种现代的显示服务器协议和相应的实现，用于替代传统的 X Window System（Xorg）。它设计的目标是更加简洁、高效、安全，并且能够提供更好的性能和更低的延迟。

  • 在传统的 Xorg 系统中，X 服务器负责处理所有显示和输入设备的管理，所有窗口和图形渲染工作都由它来完成。Xorg 的架构较为复杂，涉及多个不同的组件，因此可能存在性能瓶颈和安全隐患。

  • Wayland 的设计旨在简化这个过程，它直接将图形渲染和输入事件处理的责任分配给应用程序和合成器具体来说：

    • 合成器（Compositor）：在 Wayland 中，合成器不仅仅负责将应用程序的窗口合成到屏幕上，还负责管理显示和输入设备。Wayland 的合成器通常是直接操作图形硬件来绘制和显示内容。
    • 客户端与服务器分离：与 Xorg 不同，Wayland 中的每个应用程序（客户端）通过协议与合成器通信，直接渲染其内容，合成器负责将所有应用的图形内容合成并显示出来。这种方式减少了中间层，减少了潜在的性能瓶颈。
    • 更高效的渲染和更低的延迟：由于不需要额外的处理步骤，Wayland 在性能上通常优于 Xorg，尤其在处理高刷新率显示和更流畅的动画时，能够提供更低的延迟。
    • 更好的安全性：Wayland 也更加注重安全性，因为它避免了 Xorg 中的许多潜在的安全漏洞，例如 Xorg 允许任意应用程序查看和控制其他应用程序的内容，而在 Wayland 中，每个应用程序都只能访问自己的内容，不能随意干扰其他应用。

合成器（Compositors）

堆叠型（Stacking）

  • Enlightenment。
  • hikari：基于 wlroots 的合成器，灵感来自 cwm，积极在 FreeBSD 上开发，但也支持 Linux。
  • KDE KWin。
  • Liri Shell：Liri 的一部分，使用 QtQuick 和 QtCompositor 构建的 Wayland 合成器。
  • labwc：基于 wlroots 的合成器，灵感来自 Openbox。
  • Mutter。
  • wayfire：灵感来自 Compiz 的 3D 合成器，基于 wlroots。
  • Weston：旨在正确性、可靠性、可预测性和性能上的 Wayland 合成器。
  • wio：基于 wlroots 的合成器，旨在复制 Plan 9 的 Rio 桌面外观和感觉。

平铺型（Tiling）

  • Cagebreak：基于 cage，灵感来自 ratpoison。
  • miracle-wm：一个基于 Mir 的 Wayland 合成器，风格像 i3 和 sway，目标是比这两者更炫酷且更丰富，类似于 swayfx。
  • niri：一个支持滚动平铺的 Wayland 合成器。
  • Qtile：一款全功能的、可编程的平铺窗口管理器和 Wayland 合成器，使用 Python 编写和配置。
  • Sway：基于 wlroots 的与 i3 兼容的 Wayland 合成器。
  • SwayFx：基于 Sway 的合成器，但带有视觉效果！
  • Velox：基于 swc 的简洁窗口管理器，灵感来自 dwm 和 xmonad。

动态型（Dynamic）

  • dwl：基于 wlroots 的类似 dwm 的 Wayland 合成器。
  • Hyprland：一款动态平铺的 Wayland 合成器，注重外观与功能的平衡。
  • japokwm：一款基于 wlroots 的动态 Wayland 平铺合成器，围绕创建布局展开。
  • river：灵感来自 dwm 和 bspwm 的动态平铺 Wayland 合成器。
  • Vivarium：一款动态平铺 Wayland 合成器，基于 wlroots，桌面语义灵感来自 xmonad。

其他

  • Cage：显示单个全屏应用，如同自助终端。
  • phoc：一个为移动设备设计的轻量 wlroots 合成器。

Desktop Environment（桌面环境）

  • 文档

  • 桌面环境是一个完整的用户界面，它包括了图形界面、应用程序、工具和基本的交互元素，目的是让用户能够更容易地与系统进行交互。

  • 桌面环境通常会包括以下组件：

    • 图形用户界面（GUI）：图标、菜单、按钮、工具栏等。
    • 桌面应用程序：文件管理器、文本编辑器、终端、浏览器等。
    • 默认设置：背景、主题、窗口装饰、面板、菜单等。
    • 集成的工具：例如网络管理工具、电源管理工具等。

常用的桌面环境

  • GNOME：GNOME 桌面环境是一个吸引人且直观的桌面，具有现代 (GNOME) 和经典 (GNOME Classic) 会话。
  • KDE Plasma：KDE Plasma 桌面环境是一个熟悉的工作环境。Plasma 提供了一个现代桌面计算体验所需的所有工具，让你可以从一开始就提高生产力。
  • Xfce：Xfce 体现了传统的 UNIX 模块化和重用哲学。它由许多组件组成，提供现代桌面环境所需的所有功能，同时保持相对轻量。它们被单独打包，可以从可用的包中选择，创建最适合个人的工作环境。
  • Cinnamon：Cinnamon 致力于提供传统的用户体验。Cinnamon 是 GNOME 3 的一个分支。

其它的桌面环境

  • Budgie：Budgie 是一个为现代用户设计的桌面环境，专注于简洁与优雅。
  • COSMIC：COSMIC 是由 System76 为 Pop! OS 创建的基于 Rust 的桌面环境，灵感来自于高度自定义的 GNOME。仍处于 alpha 阶段。
  • Cutefish：Cutefish 是一个新的现代桌面环境。
  • Deepin Desktop Environment：Deepin 桌面界面和应用程序具有直观和优雅的设计。移动、共享和搜索等变得异常愉快。
  • Enlightenment：Enlightenment 桌面外壳提供一个高效的窗口管理器，基于 Enlightenment Foundation Libraries，同时还有其他基本桌面组件，如文件管理器、桌面图标和小部件。支持主题，并且能够在老旧硬件或嵌入式设备上运行。
  • GNOME Flashback：GNOME Flashback 是 GNOME 3 的一个外壳，最初被称为 GNOME 后备模式。桌面布局和底层技术与 GNOME 2 相似。
  • LXDE：Lightweight X11 Desktop Environment 是一个快速且节能的桌面环境。它具有现代化界面、多语言支持、标准键盘快捷键和其他功能，如标签文件浏览。根本上设计为轻量化，LXDE 力求比其他桌面环境更少占用 CPU 和内存。
  • LXQt：LXQt 是 LXDE 的 Qt 移植版本，也是 LXDE 即将推出的版本。它是 LXDE-Qt 和 Razor-qt 项目的合并结果：一个轻量、模块化、极速且用户友好的桌面环境。
  • MATE：MATE 提供一个直观且吸引人的桌面，采用传统的比喻。MATE 起源于 GNOME 2 的一个分支，但现在使用 GTK 3。
  • Pantheon：Pantheon 是最初为 elementary OS 发行版创建的默认桌面环境。它完全用 Vala 和 GTK3 工具包编写。就可用性和外观而言，桌面在某些方面与 GNOME Shell 和 macOS 有相似之处。
  • Phosh：Phosh 是一个基于 GNOME 的 Wayland shell 用户界面，适用于移动设备。
  • Sugar：Sugar Learning Platform 是一个由活动组成的计算机环境，旨在帮助 5 至 12 岁的儿童通过丰富的媒体表达共同学习。Sugar 是全球性努力的一部分，旨在为每个孩子提供优质教育的机会。Sugar 当前在 40 多个国家中有近一百万儿童使用，支持 25 种语言。
  • UKUI：UKUI 是一个轻量级的 Linux 桌面环境，基于 GTK 和 Qt 开发。UKUI 是 Ubuntu Kylin 的默认桌面环境。
  • CDE：Common Desktop Environment 是一个为 Unix 和 OpenVMS 提供的桌面环境，基于 Motif 小部件工具包。它曾是 UNIX98 工作站产品标准的一部分，并且长期以来是商业 Unix 工作站的“经典”桌面。尽管它是一个遗留环境，但它仍在通过对 Linux 系统的支持保持活跃。
  • EDE：Equinox Desktop Environment 是一个旨在简单、极轻量且快速的桌面环境。
  • Liri：Liri 是一个具有现代设计和功能的桌面环境。Liri 是 Hawaii、Papyros 和 Liri 项目的合并。高度实验性。
  • Lumina：Lumina 是一个为 FreeBSD 编写的轻量级桌面环境，使用 Qt 5，并采用 Fluxbox 作为窗口管理器。
  • MaXX Interactive Desktop：MaXXdesktop 从头开始设计，注重速度、响应快速、轻量简洁而非视觉效果。
  • Maui Shell：Maui Shell 是一个适用于桌面、平板和手机的融合外壳。
  • Moksha：Moksha 是 Enlightenment 的一个分支，目前是基于 Ubuntu 的 Bodhi Linux 的默认桌面环境，支持可选模块。
  • Orbitiny Desktop：Orbitiny Desktop 是一个全新的、100% 可移植的创新型桌面环境，兼具传统和现代化外观，专为 Linux 设计。
  • PaperDE：一个基于 Qt/Wayland 和 Wayfire 构建的桌面环境。
  • plainDE：一个轻量级的 GNU/Linux 桌面环境，使用 Qt 编写。目前包括面板和控制中心。
  • Plasma Mobile：Plasma Mobile 是一个基于 KDE Plasma 堆栈和 Wayland 的移动桌面环境。
  • theDesk：theDesk 是一个尽可能透明的桌面环境。它使用 Qt 5 作为其小部件工具包，KWin 作为窗口管理器。
  • Trinity：Trinity Desktop Environment (TDE) 项目是一个为类 Unix 操作系统设计的计算机桌面环境，主要目标是保留 KDE 3.5 的整体计算风格。

Window Manager（窗口管理器）

  • 窗口管理器是负责管理和控制窗口的大小、位置和行为的程序。它并不包括图形界面应用，而是直接管理窗口的显示和交互。
  • 窗口管理器控制以下内容：
    • 窗口的布局：决定窗口如何在屏幕上排列（比如层叠或平铺）。
    • 窗口的外观：包括边框、标题栏、按钮等。
    • 窗口的控制：比如最大化、最小化、关闭、调节大小等。

常用的窗口管理器

  • i3：平铺窗口管理器，完全从头开始编写。i3 是为了填补 wmii 的不足而创建的，wmii 是当时开发者最喜爱的窗口管理器，但它没有提供所需的功能（例如多显示器支持），存在一些 bug，长时间未更新，且不易修改（源代码注释/文档完全缺失）。i3 在多显示器支持和树形结构方面有所不同。
  • Openbox：高度可配置的窗口管理器，具有广泛的标准支持。box视觉风格以其简洁外观而闻名。Openbox使用box视觉风格，同时为主题开发者提供比以前的*box实现更多的选项。自2010年以来已"功能完整"，但仍在维护中。

其它的窗口管理器

浮动窗口管理器

  • 2bwm：快速的浮动窗口管理器，特点是有两个边框，基于XCB库编写，源自Michael Cardell编写的mcwm。在2bwm中，一切都可以通过键盘操作，但也可以使用指针设备来移动、调整大小以及升降窗口。
  • 9wm：受Plan 9的rio启发的X11窗口管理器。
  • AfterStep：最初基于NeXTStep界面的外观和感觉，为用户提供一致、干净、优雅的桌面。AfterStep的开发目标是提供桌面配置的灵活性，改善美学，并高效利用系统资源。
  • berry：轻量级窗口管理器，使用C语言编写。通过命令行客户端控制，可以通过像sxhkd这样的热键守护进程来控制窗口，或通过Shell脚本扩展功能。它提供可扩展的主题选项，支持双边框、标题栏和窗口文本。berry将新窗口放置在空闲空间中，并支持虚拟桌面。
  • Blackbox：轻量级的X窗口管理器，没有那些令人讨厌的库依赖。Blackbox使用C++编写，并包含完全原创的代码（尽管它的图形实现与WindowMaker类似）。
  • Compiz：使用GLX_EXT_texture_from_pixmap将重定向的顶级窗口绑定到纹理对象的OpenGL合成管理器。它具有灵活的插件系统，并设计成可以在大多数图形硬件上良好运行。
  • cwm：最初源自evilwm，后来从头重写，cwm的目标是简单，并提供诸如窗口搜索等有用的功能。
  • eggwm：轻量级的QT4/QT5窗口管理器。
  • Enlightenment：Enlightenment不仅是Linux/X11等的窗口管理器，还是一套帮助你创建美丽用户界面的库，使工作变得比传统方法更简单，避免了传统工具包的麻烦。
  • evilwm：极简主义的X窗口管理器。"极简主义"在这里并不意味着它过于简陋以至于无法使用——而是指它省略了很多让其他窗口管理器变得无法使用的东西。
  • Fluxbox：基于Blackbox 0.61.1代码的X窗口管理器。它资源占用非常轻，但却有很多功能，使其成为一个非常快速且易于使用的桌面体验。使用C++编写，并遵循MIT许可协议。
  • Flwm：尝试结合多个窗口管理器的最佳思路。主要的影响和代码基础来自Chris Cannam的wm2。
  • FVWM：功能强大的ICCCM兼容的多虚拟桌面X窗口管理器。开发仍然活跃，支持也非常好。
  • Gala：来自elementaryos的美观窗口管理器，属于Pantheon的一部分。它也是一个合成管理器，基于libmutter。
  • Goomwwm：使用C编写的X11窗口管理器，作为一个清晰的项目进行开发。它以最小的浮动布局管理窗口，并提供灵活的键盘驱动控制，用于窗口切换、调整大小、移动、标记和拼贴。它还非常快速、轻量级、无模式、支持Xinerama，并尽可能兼容EWMH。
  • IceWM：X窗口管理器。IceWM的目标是速度、简单性，并且不干扰用户。
  • jbwm：基于evilwm，最小配置文件大小约为16KB，专注于小二进制大小和可用性，结合了可选的标题栏和XFT标题栏字体渲染，作为编译时选项，jbwm还具有比evilwm更易用的键绑定。
  • JWM：X窗口管理器。JWM用C语言编写，并且仅使用最小的Xlib。
  • Karmen：X窗口管理器，由Johan Veenhuizen编写。它设计为"只要工作就好"。没有配置文件，除了Xlib以外没有其他库依赖。输入焦点模型是点击聚焦。Karmen的目标是符合ICCCM和EWMH。
  • KWin：KDE的标准窗口管理器，自KDE 4.0以来，也是一款合成管理器。这样，KWin可以提供类似Compiz的高级图形效果，同时也提供KDE早期版本的所有功能（如与KDE其余部分的良好集成、高度可配置性、防止焦点窃取、良好的错误处理等）。同时也作为Wayland的合成器。
  • lwm：一款尽量不打扰用户的X窗口管理器。没有图标、按钮栏、图标停靠栏、根菜单等：如果你需要这些，其他程序可以提供。没有配置选项：如果你想要配置，你就需要换一个窗口管理器，它会帮助你的操作系统征服硬盘空间和物理内存。
  • Marco：MATE窗口管理器，Metacity的一个分支。
  • Metacity：该窗口管理器致力于保持安静、小巧、稳定，完成工作并保持不引人注目。它用于遗留的GNOME 2和GNOME Flashback会话，已经被Mutter取代。
  • MLVWM：类Macintosh虚拟窗口管理器，基于FVWM的后代，尝试在布局和窗口设计上模拟Mac OS X之前的Macintosh外观和感觉。
  • Muffin：用于Cinnamon的窗口和合成管理器。Mutter的一个分支，基于Clutter，使用OpenGL。不能在Cinnamon之外使用。
  • Mutter：用于GNOME的窗口和合成管理器，基于Clutter，使用OpenGL。也用作Wayland合成器。
  • MWM：Motif窗口管理器（MWM），是一个基于Motif工具包的X窗口管理器。
  • pawm：X窗口系统的窗口管理器。因此，它不是一个"桌面"，也不会为你提供一大堆没用的选项，只提供运行X应用程序所需的功能，同时拥有友好且易于使用的界面。
  • PekWM：窗口管理器，曾经基于aewm++窗口管理器，但经过足够的演变，已经不再像aewm++。它有更多的功能，包括窗口分组（类似于Ion、PWM或Fluxbox）、自动属性、Xinerama、支持多重快捷键等。
  • Sawfish：可扩展的窗口管理器，使用基于Lisp的脚本语言。其策略相对于大多数窗口管理器非常简单。目标只是以最灵活和最美观的方式管理窗口。所有高级窗口管理功能都通过Lisp实现，便于未来的扩展或重新定义。
  • sowm：简单的意见窗口管理器，提供全屏切换、窗口居中和混合工作流（即鼠标和键盘）。
  • TinyWM：创建的极简窗口管理器，约50行C代码。它可能对学习创建窗口管理器的基础知识有帮助。还有一个基于python-xlib的Python版本。
  • twm：X的简单窗口管理器，自1989年以来一直是Xorg的默认/备用管理器。提供标题栏、窗体管理、几种图标管理形式、用户定义的宏功能、点击输入和指针驱动的键盘焦点，以及用户指定的键和指针按钮绑定。
  • ukwm：轻量级的GTK+窗口管理器，是UKUI桌面环境的默认窗口管理器。
  • UWM：UDE的终极窗口管理器。
  • Wind：小型浮动窗口管理器。试图遵循ICCCM、EHWM和NetWM。
  • WindowLab：小型简约窗口管理器，采用新颖设计。它具有点击聚焦但不提升聚焦政策、窗口大小调整机制，可以同时改变窗口的一个或多个边缘，并且有创新的菜单栏与任务栏共享相同的屏幕区域。
  • Window Maker：X11窗口管理器，最初设计用于为GNUstep桌面环境提供集成支持。在各个方面，它重现了NEXTSTEP用户界面的优雅外观和感觉。它快速、功能丰富、易于配置和使用。
  • WM2：X窗口管理器。它提供了一种不寻常的窗口装饰风格，且其功能最小化，以适应作者的设计理念。wm2不具备配置功能，只有通过编辑源代码并重新编译才能配置，旨在为那些不希望窗口管理器过于友好的用户设计。
  • worm：是一款动态标签窗口管理器，支持浮动布局和主堆叠布局，用Nim编写。
  • Xfwm：Xfwm 是 Xfce 桌面环境的窗口管理器，负责管理应用程序窗口在屏幕上的布局，提供美观的窗口装饰，管理工作区或虚拟桌面，并原生支持多显示器模式。它提供了自己的合成管理器（基于 X.Org Composite 扩展），支持真实的透明度和阴影效果。Xfwm 还包括一个键盘快捷键编辑器，用于用户特定命令和基本的窗口操作，并提供一个首选项对话框，允许进行高级设置。

平铺式窗口管理器

  • Bspwm：bspwm 是一个平铺式窗口管理器，它将窗口表示为一个完全二叉树的叶子节点。它支持 EWMH 和多显示器，可以通过消息进行配置和控制。
  • EXWM：EXWM（Emacs X Window Manager）是一个为 Emacs 构建的全功能平铺式 X 窗口管理器，基于 XELB。它具有完全基于键盘的操作，混合布局模式（平铺和堆叠），动态工作区支持，ICCCM/EWMH 兼容，RandR（多显示器）支持，并且内置了系统托盘。
  • Herbstluftwm：手动平铺窗口管理器，基于 Xlib 和 Glib。其布局基于将窗口分割成子框架，再将其分割或填充窗口（类似于 i3/musca）。可以动态添加/删除标签。每个标签都有自己的布局。每个显示器上只显示一个标签。标签是显示器独立的（类似于 xmonad）。它通过 ipc 调用由 herbstclient 配置。配置文件只是一个在启动时运行的脚本（类似 wmii/musca）。
  • Larswm：基于 9wm 的平铺窗口管理器。
  • LeftWM：使用 Rust 编写的平铺窗口管理器。
  • Notion：X 窗口系统的平铺式标签窗口管理器，采用“平铺”和“标签”窗口模式。
    • 平铺：你将屏幕分成不重叠的“瓷砖”。每个窗口占用一个瓷砖，并最大化到其中。
    • 标签：一个瓷砖可以包含多个窗口，它们将以标签的形式显示。
    • 静态：大多数平铺窗口管理器是“动态”的，意味着它们会根据窗口的出现或消失自动调整瓷砖的
    • 小和位置。相比之下，Notion 不会自动改变平铺。
    • Notion 是 Ion3 的一个分支。
  • Ratpoison：简单的窗口管理器，没有大型的库依赖、花哨的图形、窗口装饰和老鼠依赖。它主要受 GNU Screen 启发，配置通过简单的文本文件进行。Ratpoison 有一个信息条，仅在需要时显示。它既是应用启动器也是通知条。Ratpoison 不包括系统托盘。
  • Stumpwm：平铺、完全由键盘操作的 X11 窗口管理器，完全用 Common Lisp 编写。Stumpwm 旨在可定制且视觉上简洁。它提供了各种挂钩来附加个人定制和可调整的变量，且可以在运行时重新配置和重新加载。没有窗口装饰、图标和按钮。其信息条可以设置为常时显示或仅在需要时显示。
  • ZWM：一个最小化的平铺窗口管理器，用 C 编写，使用 XCB API 与 Xorg 服务器通信。它支持多显示器，支持 EWMH 规范的子集，且提供广泛的配置选项。

动态窗口管理器

  • awesome：高度可配置的下一代框架窗口管理器，适用于 X。它非常快速、可扩展，并且使用 GNU GPLv2 许可。通过 Lua 配置，内置系统托盘、信息条和启动器。它还提供了用 Lua 编写的扩展。awesome 使用 XCB，而不是 Xlib，这可能会提高速度。awesome 还有其他功能，比如早期替代通知守护程序、右键菜单等。

  • dk：一个基于列表的平铺窗口管理器，灵感来自 dwm、bspwm 和 xmonad。

  • dwm：动态窗口管理器，适用于 X。它在平铺、单窗口和浮动布局中管理窗口。所有布局都可以动态应用，优化当前应用和任务的环境。它不包含系统托盘或自动启动器，但与 dmenu 配合良好，因为它们来自同一作者。dwm 没有文本配置文件，配置完全通过修改 C 源代码进行，每次更改后都需要重新编译和重启。

  • echinus：简单轻量的平铺和浮动窗口管理器，起初作为 dwm 的一个分支，经过修改后成为功能完善的重父窗口管理器，支持 EWMH。它具有与 EWMH 兼容的面板/任务栏，称为 ouricoAUR。

  • FrankenWM：基本上是 monsterwm，浮动窗口做得更好。增加了的功能包括：更多布局（斐波那契、等比例堆叠、双堆叠），可调节间隙（和边框），最小化/最大化单一窗口，隐藏/显示所有窗口，单独调整主窗口和堆栈的大小，反转堆栈。

  • spectrwm：小型动态平铺窗口管理器，适用于 X11，灵感来源于 xmonad 和 dwm。它尽量避免干扰，提供更多有用的屏幕空间。它具有合理的默认配置，并使用文本文件配置。它是为黑客而写的，力求小巧、紧凑、快速。它内置了一个状态栏，用户可以通过自定义脚本进行设置。

  • Qtile：全功能的、可扩展的平铺窗口管理器，使用 Python 编写。Qtile 简单、紧凑、易于扩展。用户可以轻松编写自己的布局、部件和内置命令。Qtile 完全使用 Python 编写和配置，这意味着用户可以利用该语言的全部功能和灵活性来定制其需求。

  • Wingo：完全功能的真正混合窗口管理器，支持每显示器工作区，平铺和浮动模式可以独立切换。可以在一个工作区内使用平铺，在另一个工作区内使用浮动。Wingo 可以通过其自有的命令语言进行脚本化，完全支持主题，并支持用户定义的钩子。Wingo 使用 Go 编写，并没有运行时依赖。

  • xmonad：动态平铺 X11 窗口管理器，用 Haskell 编写和配置。在普通的窗口管理器中，用户花费大量时间对齐和寻找窗口，而 xmonad 通过自动化此过程使工作更轻松。XMonad 使用 Haskell 进行配置。所有配置更改必须重新编译 xmonad，因此必须安装 Haskell 编译器（超过 100MB）。一个名为 xmonad-contrib 的大型库提供了许多额外功能。

Display Manager（显示管理器）

  • 显示管理器是启动和管理登录会话的程序。它负责在系统启动时显示登录界面，提供用户身份验证（如用户名和密码），然后启动桌面环境或窗口管理器。
  • 显示管理器的主要功能是：
    • 登录界面：提供一个图形化或文本化的登录屏幕，供用户输入凭据。
    • 会话管理：在用户成功登录后，启动并管理桌面环境或窗口管理器。
    • 用户切换：在多个用户之间切换。

常用的显示管理器

  • LightDM：跨桌面显示管理器，可以使用任何工具包编写的各种前端。
  • SDDM：基于 QML 的显示管理器，是 KDM 的继任者；推荐用于 Plasma 和 LXQt。

其它的显示管理器

控制台显示管理器

  • CDM：使用 Bash 编写的登录管理器。
  • Console TDM：一个纯 Bash 编写的 xinit 扩展。
  • nodm：自动登录的显示管理器。自 2017 年起已停用。
  • Ly：用于 Linux 和 BSD 的 TUI（类似 ncurses）显示管理器，支持 X 和 Wayland 会话。
  • tbsm：一个 Bash 会话或应用程序启动器，支持 X 和 Wayland 会话。
  • emptty：简单的 CLI 显示管理器，支持 X 和 Wayland。
  • loginx：一个结合了 getty/login/xinit 的工具，带有友好的 curses 界面。
  • Lemurs：用 Rust 编写的 TUI 显示管理器，支持 X、Wayland 和 TTY shell 会话。

图形界面显示管理器

  • Entrance：Enlightenment 显示管理器。高度实验性，并且没有适当的 systemd 支持，已经停止开发。
  • GDM：GNOME 显示管理器。
  • LXDM：LXDE 显示管理器。可以独立于 LXDE 桌面环境使用。
  • SLiM：轻量且优雅的图形登录解决方案。自 2013 年起停止开发，不完全兼容 systemd。
  • XDM：支持 XDMCP 和主机选择的 X 显示管理器。

登录守护进程

  • greetd：支持控制台和图形界面 greeter 的登录守护进程。
